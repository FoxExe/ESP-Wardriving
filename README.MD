# ESP-12 Wardriving Tool (ESP8266)

Прошивка для вардрайвинга на базе ESP-12E/S или ESP-07S. Система сканирует Wi-Fi сети, привязывает их к GPS-координатам и сохраняет данные во внешнюю SPI Flash память.

## Аппаратная часть (Hardware)

- **ESP-12E / ESP-12S / ESP-07S** - (ESP8266) последний предпочтительнее, т.к. можно подключить внешнюю антенну
- **GPGY-NEO6MV2** Или аналогичный NMEA GPS модуль
- **Winbond 25Q64FVAIG** - 2/4/8/16MB SPI Flash - используется для хранения логов (чтобы не насиловать внутрений флеш)
- **SSD1306** i2c, 128x32, OLED дисплей. Опционально (имеется веб-интерфейс).
- **Опционально**: Мониторинг заряда АКБ через ADC0 (делитель на резисторах 100кОм + 30кОм).
- **Опционально**: Кнопка на GPIO0 (Flash/Program).

## Подключение

- **GPIO [12, 13, 14, 15]** - SPI флешка
- **GPIO [4, 5]** - Экран
- **GPIO [1, 3]** - GPS (UART0)
- **GPIO [2, -]** - UART1 - Отладочные сообщения
- **GPIO 0** - кнопка прошивки / функциональная кнопка (сейчас включает/выключает GPS)
- **ADC0** - Измеренеие уровня заряда батареи через резисторный делитель

## Внимание!

- На модулях ESP-12S/07S GPIO2 может быть вообще не подключен и использовать его невозможно!
- Если на вашем модуле не работает GPIO2, но есть светодиод (и он моргает в такт отладочным сообщениям) - можно припаяться к нему для отладки.
- Если не хочется постоянно выдёргивать GPS модуль и подключать UART для заливки прошивки - можно использовать SoftwareSerial на GPIO[15, 2], а SPI_CS перенести на GPIO16 (вероятно, понадобится подтягивающий резистор). Но SoftwareSerial работает крайне нестабильно на скоростях выше 9600. Да и забирает до 10КБ оперативки (!).

## Экран

1.  **Статус-бар:** Время, количество записанных точек с момента включения, заряд батареи.
2.  **Левый график** - Уровни сигналов спутников. Точкой помечены используемые спутники
3.  **Правый график** - Уровни сигналов Wi-Fi по каналам, с лева направо (1..13, максимумы)
4.  **Между графиками** - Иконка статуса записи, точность позиционирования (метры), использование флешка (проценты, по кол-ву блоков).

## Веб-интерфейс и настройки

- Построен на **Bootstrap 5** + **Bootstrap Icons**.
- Позволяет изменить настройки устройства
- Возможность перезагрузить устройство
- Список найденных Wi-Fi сетей (имена, уровни сигналов, шифрование и прочее)
- Вывод информации о GPS: Уровни сигналов спутников, координаты, высота, точность, время и прочее
- Список записанных блоков с возможностью скачивать и удалять выбранные блоки.

# Примечания по файлам

- [Settings.h](/src/Settings.h) - Отвечает за загрузку и сохранение настроек.  
  Настройки по умолчанию:

    ```c
    #define CFG_FILE "/settings.bin"    // Имя файла (littlefs), в котором будут храниться настройки

    struct ConfigData {
    	uint16_t version = 1;           // Версия файла конфигурации
    	char ssid[32] = "GPS_Tracker";  // Имя Wi-Fi Access Point для доступа к веб-интерфейсу
    	char pass[64] = "testtest";     // Пароль AP
    	uint16_t scan_interval = 10;	// Интервал сканирования по умолчанию, секунд
    	uint16_t min_acc = 20;			// Минимальная точность в метрах для начала сканирования
    	int16_t tz_offset = 300;		// Часовой пояс для отображения местного времени на экране (offset, минуты). Например: 300=Екатеринбург, 180=Москва
    	bool auto_tz = false;			// Автоопределение часового пояса по текущему азимуту (неточно)
    	bool rotate_logs = true;        // При заполнени флеша будут перезаписываться наиболее старые блоки
    };
    ```

- [Battery.h](/src/Battery.h) - Код монитора батареи.
    ```c
    #define TURNOFF_THRESHOLD 1  // Порог заряда батареи (процентов) для перехода в спящий режим (отключение устройства)
    #define BATT_DIVIDER_K 5.1   // Коэффицент резисторного делителя для точной настройки монитора напряжения батареи. ~5.5 для ESP-12E, ~5.1 для ESP-07S
    ```
- [log_reader.py](/log_reader.py) - Пример кода для "распаковки" дампов логов (_block_xxx.bin_)
- [extra.py](/extra.py) - Скрипт для Platformio, сжимает файлы веб-интерфейса в .gz для экономии места

# Заметки:

- SoftwareSerial был выбран исключительно для удобства разработки (прошивки) устройства. По факту можно подключить GPS модуль к UART0, а отладку вывести на GPIO2 (UART1_Tx). К томуже тогда можно будет перевести GPS модуль на более высокую скорость работы и частоту опроса. (SW Serial потребляет кучу RAM и CPU, ещё и работает крайне нестабильно на более высоких скоростях).
- SPI1_CS переведён на GPIO16 ровно по той-же причине - нужен был GPIO15 ради прерываний для SW Serial (16й их не поддерживает)
- ESP-07S с антенной от роутера показывает -35..40dbm через стену от роутера.  
  ESP-12S/E со встроенной антенной тамже показывает -45..50dbm.  
  Но на открытой местности, на горе, при прямой видимости, некоторые "радиомосты" и прочие "направленные антенны" слышат одинаково далеко (2-3км) с одинаковым уровнем сигнала (~ -90dbm +/-2).
- Внешняя флеш нужна только для сохранения логов, чтобы при износе её можно было легко заменить.  
  По-хорошему надо бы использовать Micro-SD и простой текстовой лог. Намного удобнее и проще, но у меня банально не нашлось нужного разъёма под рукой (Впрочем, как и свободной SD карточки), а вот SPI флешек от старых материнских плат набралось с десяток.
- Формат записи логов на флеше:  
  Флеш разделён на блоки по 64кб  
  Начало каждого блока - заголовок "0xDDCC"  
  Затем записывается структура с GPS данными и кол-вом найденных WiFi. После неё - блоки RSSI+Offset от начала блока, указывающий на структуру с информацией о Wi-Fi устройстве. Это сделано для экономии памяти из-за повторяющихся Wifi.  
  Паралельно (вернее, до записи GPS) с конца блока записываются структуры с информацией о WiFi (номер канала, тип шифрования, длинна SSID). А вот сам SSID записывается ПЕРЕД структурой. Это сделано для того, чтобы можно было вычитывать информацию о WiFi с конца (читаем структуру, узнаём длину ssid, вычитываем ssid, переходим к следующей структуре wifi).  
  Так данные пишутся навстречу друг-другу, пока блок не заполнится полностью, после чего перехоим к следующему блоку.  
  Чтобы не искать каждый раз AP Offset - держим в памяти "кеш" MAC->Offset.  
  Если MAC совпал, но изменились данные (канал, шифрование, имя) - будет созданна новая запись. Это можно использовать для отслеживания истории изменения этих данных со временем. (дату изменения придётся брать из GPS точки, в которой засветились изменения в первый раз).
- 8MB флеша хватит на 1-3 дня логов "нон-стоп". Сильно зависит от скорости перемещения и "плотности" эфира (кол-ва уникальных Wifi).
- Одного аккумулятора 18650 на 3200mAh хватает на 15-20 часов работы (с учётом крайне неэффективного преобразования 3.7v -> 5v -> 3.3v (AMS1117-3.3), который я использую)
- В коде используется усреднение координатной точки для лучшей точности замеров (т.к. на сканирование WiFi уходит порядка 2-3 секунд, за которые можно проехать приличное расстояние, если на машине)
